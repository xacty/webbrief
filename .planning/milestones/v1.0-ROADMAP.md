## Overview

Refactor sistemático del sistema visual de WeBrief contra el framework Refactoring UI (7 principios). El milestone v1.0 entrega: design tokens completos, librería de componentes compartidos, y migración por área (admin/auth → editor → públicas) con score promedio 8.5/10. Sin cambios funcionales, sin regresiones, sin migración de stack (CSS Modules + variables CSS preservados).

**Milestone Goal achieved:** Sistema visual unificado con tokens completos, componentes reutilizables y score promedio 8.5/10 en Refactoring UI sobre todas las áreas (admin, auth, editor, públicas), sin regresiones funcionales.

## Milestone Summary

**Decimal Phases:** None (no insertions during milestone)

**Key Decisions:**

- Refactor en branch `refactor/ui-system` aislada (worktree) — Permite trabajo en paralelo sin tocar dev server del usuario; merge controlado
- Bootstrap GSD mínimo (sin ingest completo) — 12 docs narrativos del repo arrastrarían fases pasadas (security, db, mcp); enfoque solo en refactor UI
- Mantener CSS Modules en lugar de migrar a Tailwind — Minimiza disrupción; 7,500 líneas de CSS no se reescriben en un milestone
- Aplicar framework Refactoring UI (Adam Wathan/Steve Schoger) como guía — Sistema opinionated y comprobado; 7 principios concretos vs juicio subjetivo
- Editor sub-tokens (`--wb-editor-*`, `--wb-tooltip-*`, `--wb-comment-*`, `--wb-section-*`) — Preserva la paleta dark del editor sin paleta paralela; consume tokens base de Phase 1
- Native controls preserved (textarea/file/radio/checkbox) — Phase 2 sin primitive equivalente; tokenizar in-place via CSS modules
- Advisory verdict para UI-09 sub-thresholds — Spacing/Typography/Color editor gaps clusterizan en Phase 4; tracked como TODOs v1.1 vs blocking gate

**Issues Resolved:**

- 7,500 líneas de CSS en 20 `.module.css` con tokens parciales → 1,643 referencias `var(--wb-*)` milestone-wide
- Editor paleta paralela hardcoded (`#212222 / #2a2a2a / #d9d9d9`) → eliminada
- Modales duplicados con z-index inconsistente entre páginas → unificados via shared `<Modal>` con `var(--wb-z-modal)`
- Z-index ad-hoc (20/50/80/120/200/300/500/1000/1200/1400/3000/9999) → tokens scale (base/dropdown/sticky/overlay/modal/popover/tooltip/toast)
- 43 hex literals en public pages → 0 outside justified `@media print`
- BriefPage success icon contrast (4.0:1) → success-700 4.5:1 (WCAG AA)

**Issues Deferred (advisory tech debt):**

- UI-09 Spacing & Sizing 7.5/10 (vs 9.0 min) — ~288 raw `padding|margin|gap` px en 9 editor CSS files; tracked in `.planning/todos/pending/001-fix-ui-spacing-editor.md`
