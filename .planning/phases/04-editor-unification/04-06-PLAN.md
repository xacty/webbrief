The migration still applies — the modal still gets `<Modal>` chrome (overlay + card + header + close + body) — but the body content is the existing image-export form, NOT an audience picker. The eyebrow text stays as-is. UI-SPEC's mention of "designer/dev audience" inside the export modal does NOT apply to this code path.

```jsx
export default function Modal({
  open,                      // boolean — controls visibility
  onClose,                   // () => void — called on Escape, overlay click, close-button click
  title,                     // string — rendered as <h2> in the header (use this for the modal title; mutually exclusive with ariaLabel for naming)
  ariaLabel,                 // string — used when title is absent (icon-only / custom-header dialogs)
  size = 'md',               // 'sm' | 'md' | 'lg' | 'xl' | 'full'  → 400 / 500 / 640 / 768 / 100% max-width
  closeOnEscape = true,      // boolean
  closeOnBackdrop = true,    // boolean — pass false to disable click-outside close (verify behavior of the original)
  showCloseButton = true,    // boolean — header X button
  initialFocusRef,           // React.RefObject — focus this ref on open (e.g., main CTA, first input)
  footer,                    // ReactNode — rendered as <footer> below body
  ariaDescribedBy,           // string — id of body element to set aria-describedby
  className,                 // string — additional class on the card
  overlayClassName,          // string — additional class on the overlay
  children,                  // ReactNode — body content
})
```
