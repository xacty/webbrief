<objective>
Tokenize every `z-index: NNN` declaration in the editor CSS modules. **This is Step 7 of 9** of the UI-SPEC migration sequence and the only plan that satisfies requirement **UI-07** (z-index unified).

Purpose: Replace the current 19 raw values (`1, 2, 10, 20, 50, 70, 80, 100, 120, 200, 300, 500, 1000, 1100, 1200, 1400, 1500, 1600, 9999`) with the Phase 1 z-index tokens (`--wb-z-base` 1, `--wb-z-dropdown` 100, `--wb-z-sticky` 200, `--wb-z-overlay` 900, `--wb-z-modal` 1000, `--wb-z-popover` 1100, `--wb-z-tooltip` 1200) plus the `EditorContextMenu` calc offset (`calc(var(--wb-z-popover) + 1)`).

Output: Eight editor CSS modules (5 page-level + 3 component-level) declare z-index exclusively via `var(--wb-z-*)`. Visual stacking is preserved.

> **Important — UI-SPEC deviation note:** UI-SPEC §"Z-index — promotes existing arbitrary values to tokens" prescribes that **all editor sticky layers collapse to `var(--wb-z-sticky)` (200) — single value**. If during execution this collapse causes a real stacking bug (e.g., toolbar dropdown clipped under sections panel), the fix is to **introduce `--wb-z-sticky-elevated: 250` in `tokens.css` and consume it for the offending element**, NOT revert to a literal numeric value. Any such introduction is documented in the SUMMARY.
</objective>
