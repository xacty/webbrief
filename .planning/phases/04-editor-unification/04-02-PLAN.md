<objective>
Migrate `frontend/src/pages/ProjectEditor.module.css` (1769 lines, ~200 hex literals) to consume Phase 4 sub-tokens (added in `04-01`) and Phase 1 global tokens. **This is Step 2 of 9** in the UI-SPEC migration sequence and is the highest-risk file in Phase 4 because it carries the editor's parallel dark palette.

Output: `ProjectEditor.module.css` with zero non-documented hex literals (the documented `rgb(255 255 255 / NN%)` and `rgb(0 0 0 / NN%)` alpha overlays stay as literals per UI-SPEC §Deviations note (a)).

This plan has a `human-verify` checkpoint after the migration because the editor is the highest-traffic surface in the app and the invariant set is dense — visual diff cannot be fully automated.
