### `Input.jsx` contract (UI-SPEC §"2. Input"):

- `import React, { forwardRef, useId, useState } from 'react';`
- `import { Eye, EyeOff } from 'lucide-react';`
- `import cn from './cn.js';`
- `import styles from './Input.module.css';`
- Default export: `const Input = forwardRef(function Input(props, ref) { ... });`
- Props (defaults shown):
  - `type = 'text'` (one of `'text' | 'email' | 'password' | 'number' | 'search' | 'tel' | 'url'`)
  - `label`, `helperText`, `error` (`error` is `boolean | string` — string overrides `helperText` and turns it red)
  - `icon = null`, `iconPosition = 'left'`
  - `fullWidth = true`, `required = false`, `disabled = false`
  - `id` (auto-generated via `useId()` when omitted)
  - Standard form props passed through: `value, defaultValue, onChange, onBlur, placeholder, name, autoComplete, ...rest`
- Behaviour:
  - `const reactId = useId(); const inputId = id ?? reactId;`
  - Helper/error text gets a deterministic id `${inputId}-helper` and is wired via `aria-describedby` only when text is present.
  - When `error` is a non-empty string: render that string; when `error === true` and no helperText: render the Spanish fallback `"Campo inválido"` (UI-SPEC §"Copywriting Contract").
  - `aria-invalid={error ? 'true' : undefined}` on the underlying `<input>`.
  - Required marker: when `required`, append `<span aria-hidden="true" className={styles.required}>*</span>` next to the label text. The `<input>` itself uses `aria-required="true"` (UI-SPEC §"Copywriting").
  - **Password toggle:** when `type === 'password'`, manage internal `const [revealed, setRevealed] = useState(false);` and render an eye toggle button on the right. The actual `<input type>` becomes `revealed ? 'text' : 'password'`. Toggle button is a `<button type="button" aria-label="{revealed ? 'Ocultar contraseña' : 'Mostrar contraseña'}" aria-pressed={revealed} className={styles.passwordToggle} onMouseDown={(e) => e.preventDefault()}>` rendering `<EyeOff>` or `<Eye>` (16px). The `onMouseDown preventDefault` keeps the input focused so caret position is preserved (matches CONTEXT.min.md `Recent Fixes`: "password visibility toggle (Eye/EyeOff from lucide-react) added to both Login and SetPassword pages").
  - **Icon precedence vs password toggle:** if `type === 'password'`, force `iconPosition = 'left'` internally (the right slot is reserved for the toggle). When the caller passes `iconPosition='right'` AND `type='password'`, ignore the prop value and emit `console.warn('[Input] iconPosition=right is ignored when type=password (eye toggle reserves the right slot)')` once.
  - Root structure (semantic):
    ```
    <div className={cn(styles.field, fullWidth && styles.fullWidth, error && styles.hasError, className)}>
      {label && <label htmlFor={inputId} className={styles.label}>{label}{required && <span aria-hidden="true" className={styles.required}>*</span>}</label>}
      <div className={cn(styles.inputWrapper, icon && styles[`hasIcon_${effectiveIconPosition}`], type === 'password' && styles.hasPasswordToggle)}>
        {icon && <span aria-hidden="true" className={cn(styles.icon, styles[`icon_${effectiveIconPosition}`])}>{icon}</span>}
        <input ref={ref} id={inputId} type={effectiveType} disabled={disabled} required={required} aria-required={required ? 'true' : undefined} aria-invalid={error ? 'true' : undefined} aria-describedby={describedBy} className={styles.input} {...rest} />
        {type === 'password' && <button type="button" className={styles.passwordToggle} aria-label={...} aria-pressed={revealed} onMouseDown={(e) => e.preventDefault()} onClick={() => setRevealed((v) => !v)}>{revealed ? <EyeOff size={16} /> : <Eye size={16} />}</button>}
      </div>
      {(helperOrError) && <span id={`${inputId}-helper`} className={cn(styles.helper, error && styles.helperError)}>{helperOrError}</span>}
    </div>
    ```
- `forwardRef` exposes the underlying `<input>` (NOT the wrapper div), per UI-SPEC §"2. Input" final paragraph.

### `Select.jsx` contract (UI-SPEC §"3. Select"):

- `import React, { forwardRef, useId } from 'react';`
- `import cn from './cn.js';`
- `import styles from './Select.module.css';`
- Default export: `const Select = forwardRef(function Select(props, ref) { ... });`
- Props:
  - `label`, `helperText`, `error`, `required = false`, `disabled = false`, `fullWidth = true`, `id`
  - `placeholder` (string — when provided AND no `value`/`defaultValue` matches, render a leading `<option disabled value="">{placeholder}</option>`; if `placeholder` is omitted, fall back to the Spanish UI-SPEC default `"Selecciona una opción"` ONLY if a placeholder slot is present, i.e. when no children/options have a default value matching the field's value).
  - `value`, `defaultValue`, `onChange`, `name`, `...rest`
  - `options` (optional `Array<{ value, label, disabled? }>`) — UI-SPEC §"3. Select" rule: "if both children and options are passed, **options wins**".
  - `children` — optional `<option>` / `<Select.Option>` nodes.
- Static property: attach `Select.Option = function Option(props) { return <option {...props}>{props.children}</option>; };` so callers can write `<Select.Option value="a">A</Select.Option>` ergonomically.
- Behaviour:
  - `const reactId = useId(); const selectId = id ?? reactId;`
  - Helper/error wiring identical to Input (`aria-describedby={`${selectId}-helper`}`, fallback string `"Campo inválido"`, `aria-invalid='true'` on error).
  - Render structure:
    ```
    <div className={cn(styles.field, fullWidth && styles.fullWidth, error && styles.hasError, className)}>
      {label && <label htmlFor={selectId} className={styles.label}>{label}{required && <span aria-hidden="true" className={styles.required}>*</span>}</label>}
      <select ref={ref} id={selectId} name={name} value={value} defaultValue={defaultValue} onChange={onChange} disabled={disabled} required={required} aria-required={required ? 'true' : undefined} aria-invalid={error ? 'true' : undefined} aria-describedby={describedBy} className={styles.select} {...rest}>
        {placeholder !== undefined && <option value="" disabled hidden>{placeholder}</option>}
        {options ? options.map(o => <option key={String(o.value)} value={o.value} disabled={o.disabled}>{o.label}</option>) : children}
      </select>
      {(helperOrError) && <span id={`${selectId}-helper`} className={cn(styles.helper, error && styles.helperError)}>{helperOrError}</span>}
    </div>
    ```
- Forward ref to the underlying `<select>`.
- DO NOT redeclare `--wb-select-chevron` / `--wb-select-padding-right` / `--wb-select-chevron-inset` / `--wb-select-chevron-size` here — base.css owns them and the global `select:not([multiple])` rule already paints the chevron. Verify by reading `frontend/src/styles/base.css` lines 27–37.
