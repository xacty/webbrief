### `Card.jsx` contract (UI-SPEC §"5. Card"):

- `import React from 'react';`
- `import cn from './cn.js';`
- `import styles from './Card.module.css';`
- Default export (NOT forwardRef per UI-SPEC §5):
  ```js
  export default function Card({
    as: As = 'div',
    padding = 'md',
    shadow = 'sm',
    radius = 'lg',
    interactive = false,
    className,
    children,
    onClick,
    ...rest
  }) {
    const isInteractive = interactive || As === 'button';
    return (
      <As
        className={cn(
          styles.card,
          styles[`padding_${padding}`],
          styles[`shadow_${shadow}`],
          styles[`radius_${radius}`],
          isInteractive && styles.interactive,
          className
        )}
        onClick={onClick}
        {...(As === 'button' ? { type: rest.type ?? 'button' } : null)}
        {...rest}
      >
        {children}
      </As>
    );
  }
  ```
- Polymorphic via `as` prop. Supported: `'div' | 'section' | 'article' | 'button'`.
- When `as === 'button'` AND no `type` was passed, default to `type='button'` so the card never accidentally submits a form.
- `interactive` enables hover/focus styles even when `as` is not `'button'`.

### `Modal.jsx` contract (UI-SPEC §"4. Modal"):

- Imports:
  ```js
  import React, { useEffect, useId, useLayoutEffect, useRef } from 'react';
  import { createPortal } from 'react-dom';
  import { X } from 'lucide-react';
  import cn from './cn.js';
  import styles from './Modal.module.css';
  ```
- Module-level body-scroll refcount (UI-SPEC §"4. Modal" Behaviour):
  ```js
  let openModalCount = 0;
  let savedBodyOverflow = '';
  function lockBodyScroll() {
    if (typeof document === 'undefined') return;
    if (openModalCount === 0) {
      savedBodyOverflow = document.body.style.overflow;
      document.body.style.overflow = 'hidden';
    }
    openModalCount += 1;
  }
  function unlockBodyScroll() {
    if (typeof document === 'undefined') return;
    openModalCount = Math.max(0, openModalCount - 1);
    if (openModalCount === 0) {
      document.body.style.overflow = savedBodyOverflow;
      savedBodyOverflow = '';
    }
  }
  ```
- Default export `Modal`:
  ```js
  export default function Modal({
    open,
    onClose,
    title,
    size = 'md',
    closeOnEscape = true,
    closeOnBackdrop = true,
    showCloseButton = true,
    initialFocusRef,
    footer,
    ariaDescribedBy,
    className,
    overlayClassName,
    children,
  }) { ... }
  ```
- Behaviour (when `open === true`):
  - Generate `const titleId = useId();` (only used as `aria-labelledby` if a title is rendered).
  - `cardRef = useRef(null);` and `previousActiveRef = useRef(null);`
  - `useLayoutEffect` on open: capture `previousActiveRef.current = document.activeElement;` then schedule (in a microtask or `requestAnimationFrame`) focus assignment — focus `initialFocusRef?.current` if provided, otherwise the first focusable descendant of `cardRef.current` (querySelector of `'a, button, [tabindex]:not([tabindex="-1"]), input, select, textarea'`), else fall back to `cardRef.current` itself with `tabIndex={-1}`. On close (cleanup), restore focus: `previousActiveRef.current?.focus?.();`.
  - `useEffect` on open: `lockBodyScroll()` and return cleanup `unlockBodyScroll()`.
  - `useEffect` on open: bind `keydown` listener — if `e.key === 'Escape' && closeOnEscape`, call `onClose()`. Bind a focus-trap on Tab: when active element is the last focusable in cardRef and `Tab` (no Shift), prevent default and focus the first; mirror for `Shift+Tab` from the first focusable.
  - **Mousedown→mouseup overlay-close pattern** (UI-SPEC §"4. Modal": "Mousedown-on-overlay + mouseup-on-overlay (NOT a single click)"). Track `const downOnOverlayRef = useRef(false);`. On the overlay's `onMouseDown`, set `downOnOverlayRef.current = (e.target === e.currentTarget);`. On its `onMouseUp`, if `closeOnBackdrop && downOnOverlayRef.current && e.target === e.currentTarget`, call `onClose()`. Reset `downOnOverlayRef.current = false` either way. Do NOT attach a plain `onClick` to the overlay.
  - Render structure (only when `open === true`):
    ```jsx
    return createPortal(
      <div
        className={cn(styles.overlay, overlayClassName)}
        onMouseDown={handleOverlayMouseDown}
        onMouseUp={handleOverlayMouseUp}
      >
        <div
          ref={cardRef}
          role="dialog"
          aria-modal="true"
          aria-labelledby={title ? titleId : undefined}
          aria-describedby={ariaDescribedBy}
          tabIndex={-1}
          className={cn(styles.card, styles[`size_${size}`], className)}
          onMouseDown={(e) => e.stopPropagation()}
        >
          {(title || showCloseButton) && (
            <header className={styles.header}>
              {title && <h2 id={titleId} className={styles.title}>{title}</h2>}
              {showCloseButton && (
                <button
                  type="button"
                  aria-label="Cerrar"
                  className={styles.closeButton}
                  onClick={onClose}
                >
                  <X size={20} aria-hidden="true" />
                </button>
              )}
            </header>
          )}
          <div className={styles.body}>{children}</div>
          {footer && <footer className={styles.footer}>{footer}</footer>}
        </div>
      </div>,
      document.body
    );
    ```
  - When `open === false`, return `null` BEFORE any portal call (no DOM cost).

- Important: stop event propagation on the card's `onMouseDown` so a mousedown that originates inside the card never bubbles to the overlay's mousedown and accidentally arms the close. Combined with the `e.target === e.currentTarget` checks, this gives the documented drag-safe close behaviour.
