### `Button.jsx` contract (locked by UI-SPEC §"1. Button"):

- `import React, { forwardRef } from 'react';`
- `import { Loader2 } from 'lucide-react';`
- `import cn from './cn.js';`
- `import styles from './Button.module.css';`
- Default export: `const Button = forwardRef(function Button(props, ref) { ... });`
- Props (defaults shown):
  - `variant = 'primary'` (one of `'primary' | 'secondary' | 'ghost' | 'danger'`)
  - `size = 'md'` (one of `'sm' | 'md' | 'lg'`)
  - `type = 'button'` (one of `'button' | 'submit' | 'reset'`)
  - `disabled = false`, `loading = false`, `fullWidth = false`
  - `icon = null` (ReactNode)
  - `iconPosition = 'left'` (one of `'left' | 'right'`)
  - `onClick`, `className`, `children`, `...rest`
- Behaviour:
  - `loading === true` implies `disabled === true` on the rendered `<button>`.
  - When `loading`, render `<Loader2>` spinner (size matches the button-size icon size: 14/16/18 for sm/md/lg) and wrap the children in a `<span className={styles.label} aria-hidden={loading || undefined}>` so its width persists but is visually hidden via `visibility: hidden` while loading. Add a `<span className={styles.srOnly}>Cargando…</span>` for screen readers (matches UI-SPEC §"Copywriting Contract": Spanish "Cargando…").
  - Icon-only support: if `children` is empty AND `aria-label` is missing AND `loading` is false, emit `console.warn('[Button] icon-only buttons require either children or aria-label')` ONCE per render (use a `useRef` boolean if needed; OR a module-level `Set` keyed by component instance — pick whichever is simpler).
  - `iconPosition='right'` swaps DOM order: label span first, icon second.
  - `cn(styles.btn, styles[`variant_${variant}`], styles[`size_${size}`], loading && styles.loading, fullWidth && styles.fullWidth, className)` for the root class.
  - Forward `ref` to the `<button>` element. Spread `...rest` onto `<button>` AFTER setting `type`, `disabled`, `onClick`, and the composed `className` so callers can NOT override those four (preserves accessibility/loading invariants).

### `Button.module.css` contract (per UI-SPEC §"Color" per-variant token map and §"Component API Contracts" §Button geometry table):

Top-level `.btn`:
```
display: inline-flex;
align-items: center;
justify-content: center;
gap: var(--wb-space-2);
border: 1px solid transparent;
border-radius: var(--wb-radius-2); /* sm/md; lg overrides to --wb-radius-3 */
font-family: var(--wb-font-sans);
font-weight: var(--wb-weight-medium);
cursor: pointer;
transition: background-color 150ms ease, border-color 150ms ease, color 150ms ease;
-webkit-appearance: none;
text-decoration: none;
white-space: nowrap;
```

Size modifiers (height fixed at 32/40/48 via documented token arithmetic; padding-x via space tokens):
- `.size_sm`: `height: var(--wb-space-8); padding: 0 var(--wb-space-3); font-size: var(--wb-text-sm); line-height: var(--wb-leading-sm); border-radius: var(--wb-radius-2);` (gap shrinks to `var(--wb-space-1)` for icon-to-label tightness — per UI-SPEC §Spacing Scale row 1).
- `.size_md`: `height: calc(var(--wb-space-8) + var(--wb-space-2)); padding: 0 var(--wb-space-4); font-size: var(--wb-text-sm); line-height: var(--wb-leading-sm); border-radius: var(--wb-radius-2);`
- `.size_lg`: `height: var(--wb-space-12); padding: 0 var(--wb-space-5); font-size: var(--wb-text-base); font-weight: var(--wb-weight-semibold); line-height: var(--wb-leading-base); border-radius: var(--wb-radius-3);`

Variant modifiers (locked by UI-SPEC per-variant token map — copy verbatim):
- `.variant_primary`: `background: var(--wb-color-primary-900); color: #ffffff;` and `:hover { background: var(--wb-color-primary-500); }`.
- `.variant_secondary`: `background: var(--wb-color-neutral-100); color: var(--wb-color-neutral-800);` and `:hover { background: var(--wb-color-neutral-200); }`.
- `.variant_ghost`: `background: transparent; color: var(--wb-color-neutral-700);` and `:hover { background: var(--wb-color-neutral-100); }`.
- `.variant_danger`: `background: var(--wb-color-danger-600); color: #ffffff;` and `:hover { background: var(--wb-color-danger-700); }`.

Disabled (single rule shared across variants):
```
.btn:disabled {
  opacity: 0.5;
  cursor: not-allowed;
}
.btn:disabled:hover {
  background: inherit;
}
```
Note: under disabled, the variant's own background should resurface — implement by listing the per-variant `:hover` rules with a `:not(:disabled):hover` selector instead of plain `:hover`.

Focus ring (UI-SPEC §"Color" Accent reserved): `.btn:focus-visible { outline: none; box-shadow: 0 0 0 3px var(--wb-color-primary-200); }`. NEVER `outline: none` without the replacement ring — accessibility baseline.

Active state: `:active { transform: translateY(0); }` and rely on darker hover bg (UI-SPEC explicitly says no Y translation).

Loading state — keeps width stable:
```
.loading .label { visibility: hidden; }
.loading .spinner { position: absolute; }
.btn.loading { position: relative; }
.spinner { animation: spin 1s linear infinite; }
@keyframes spin { from { transform: rotate(0); } to { transform: rotate(360deg); } }
```

`fullWidth`: `.fullWidth { width: 100%; }`.

`srOnly` (visually-hidden screen-reader text, used for the "Cargando…" label):
```
.srOnly {
  position: absolute;
  width: 1px;
  height: 1px;
  padding: 0;
  margin: -1px;
  overflow: hidden;
  clip: rect(0 0 0 0);
  white-space: nowrap;
  border: 0;
}
```

Reduced motion (mandatory per Phase 1 base.css pattern + UI-SPEC §Accessibility Baseline):
```
@media (prefers-reduced-motion: reduce) {
  .btn { transition: none; }
  .spinner { animation: none; }
}
```

PROHIBITED in this CSS module:
- Any `text-transform: uppercase` declaration.
- Any raw hex colour (e.g., `#fff`, `#dc2626`) — `#ffffff` is allowed ONLY for the locked Button-primary/danger text color per the UI-SPEC token map; nothing else.
- Any raw px value for padding, gap, border-radius EXCEPT the documented height arithmetic from UI-SPEC §Spacing Scale (sizes 32/40/48 derived via `var(--wb-space-8)`, `calc(var(--wb-space-8) + var(--wb-space-2))`, `var(--wb-space-12)`).

### `Badge.jsx` contract (UI-SPEC §"6. Badge"):

- `import React from 'react';`
- `import cn from './cn.js';`
- `import styles from './Badge.module.css';`
- Default export: `function Badge({ variant = 'neutral', size = 'md', icon, className, children, ...rest }) { ... }`. NO forwardRef (UI-SPEC does not require it; only Button/Input/Select/Modal do).
- Variants: `'neutral' | 'success' | 'warning' | 'danger'`.
- Sizes: `'sm' | 'md'`.
- Render structure: `<span className={cn(styles.badge, styles[`variant_${variant}`], styles[`size_${size}`], className)} {...rest}>{icon && <span className={styles.icon} aria-hidden="true">{icon}</span>}{children}</span>`.
- Pure presentational — do NOT add `role="status"` by default (UI-SPEC: "caller adds it for live regions"). Caller passes via `...rest` if needed.
- If only icon is provided (no children), no `aria-label` enforcement here — UI-SPEC §"Copywriting Contract" says caller MUST pass `aria-label`. Do not console.warn here (Badge is purely presentational; warning lives in Button only).

### `Badge.module.css` contract (UI-SPEC §"6. Badge" + §"Color"):

```
font-weight: var(--wb-weight-medium);

letter-spacing: 0;
white-space: nowrap;
```

Size modifiers:
- `.size_md`: inherits the base above (no overrides needed — base is the md size).
- `.size_sm`: `padding: 2px var(--wb-space-2);` (UI-SPEC §"6. Badge" geometry explicitly calls out 2px for sm vertical padding — this is the ONE documented raw-px exception in this module; annotate with a CSS comment `/* sm: 2px per UI-SPEC §Badge geometry */`).

Icon slot:
