| Browser → React app | URL hash carrying invite/reset tokens enters SetPassword via `onAuthStateChange` + `getSession` |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | Login form (`<Input type="password">`) | accept | Refactor preserves the existing Supabase password flow byte-for-byte; no new attack surface introduced. Shared `<Input>` does not log values. |
| T-03-02 | Information disclosure | SetPassword reset/invite token in URL hash | accept | Existing flow preserved — token never persisted in component state beyond Supabase's session handling. Refactor changes only JSX/CSS. |
| T-03-03 | Spoofing | "Olvidé mi contraseña" → `resetPasswordForEmail` | accept | No flow change; Supabase rate-limits the email-send endpoint. CONTEXT.min.md §"backend.security" notes login/reset are Supabase-direct. |
