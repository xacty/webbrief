After the palette block from Task 1, **rewrite** each legacy color token line so it points at a new `--wb-color-*` token whose value is exactly equal to the old literal — OR keep the literal in place if no clean canonical mapping exists. Both approaches preserve back-compat. Use these mappings:

```css
  /* Legacy aliases — preserve current computed values for back-compat.
     Plans 02-05 of this milestone will gradually migrate consumers
     to the new --wb-color-* tokens; do NOT remove these aliases yet. */
  --wb-bg: var(--wb-color-neutral-50);            /* was #f8fafc */
  --wb-surface: #ffffff;                          /* no canonical needed */
  --wb-surface-muted: var(--wb-color-neutral-100);/* was #f1f5f9 */
  --wb-border: #dbe3f0;                           /* keep literal — between 200 and 300 */
  --wb-border-strong: #c7d2e5;                    /* keep literal */
  --wb-text: var(--wb-color-primary-900);         /* was #091223 */
  --wb-text-muted: var(--wb-color-neutral-500);   /* was #64748b */
  --wb-primary: var(--wb-color-primary-900);      /* was #091223 */
  --wb-primary-hover: var(--wb-color-primary-500);/* was #1e293b */
  --wb-primary-soft: var(--wb-color-neutral-200); /* was #e2e8f0 */
  --wb-success: #0f766e;                          /* keep literal — Phase 1 preserves teal accent */
  --wb-danger: var(--wb-color-danger-600);        /* was #dc2626 */
```

Critical rules:
- The right-hand side of every existing token must compute to the SAME 6-digit hex value it had before this plan. If `var(...)` resolves to a different hex, fix the mapping or fall back to the literal.
- Do NOT delete `--wb-shadow-sm`, `--wb-shadow-lg`, `--wb-radius-sm`, `--wb-radius-md`, `--wb-radius-lg`, or `--wb-content-width` — Plan 02 owns those. Leave their lines untouched here.
- Keep one blank line between the palette block (Task 1) and the legacy block (Task 2) for readability.
