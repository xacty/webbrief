<task type="auto">
  <name>Task 1: Migrate one section of AccountSettingsPage.module.css to consume new tokens</name>
  <files>frontend/src/pages/AccountSettingsPage.module.css</files>
  <action>
Open `frontend/src/pages/AccountSettingsPage.module.css`. Find at least 2 hardcoded numeric or color literals (e.g., `font-size: 13px`, `gap: 22px`, `padding: 10px`, `top: 24px`, a hardcoded `#hex`) and replace them with the equivalent new token from `frontend/src/styles/tokens.css`. Choose replacements where the new token's value matches the old literal exactly so the visual output is unchanged.

Suggested low-risk replacements (verify each value matches):
- `gap: 22px;` in `.page` → keep 22px (no exact token match) OR pick `gap: var(--wb-space-6); /* 24px, was 22 */` ONLY if you accept the +2px delta. Default: skip this one.
- `font-size: 13px;` in `.eyebrow` → no exact match (nearest: `--wb-text-xs` = 12px). Skip if exact-match required, or accept delta and document.
- `top: 24px;` in `.settingsNav` → `top: var(--wb-space-6);` (24px = exact match, safe).
- `padding: 10px;` in `.settingsNav` → `padding: var(--wb-space-3); /* was 10px, now 12px */` only if delta acceptable; otherwise skip.

**Pick at least 2 changes that are exact matches** (no visual delta). The token system is rich enough to make at least these safe:
- Any `24px` → `var(--wb-space-6)`
- Any `16px` → `var(--wb-space-4)`
- Any `12px` → `var(--wb-space-3)`
- Any `8px` → `var(--wb-space-2)`
- Any `4px` → `var(--wb-space-1)`
- A hardcoded `#374151` (gray text) → `var(--wb-color-neutral-700)` (4.5:1 AA)

If no opportunity for a color-token swap exists in the pilot, ADD one new declaration that uses `--wb-color-neutral-700` for an existing text color, replacing a `var(--wb-text-muted)` or a hardcoded gray, only where it does not change the rendered hex.

**Hard rule**: every replacement must be value-equivalent (same computed string) so there is zero visual diff. If there is no exact-match opportunity, document the gap in the SUMMARY rather than introducing a regression.

After editing, the file MUST contain at least one occurrence of `var(--wb-color-` AND at least one occurrence of `var(--wb-space-` (or `--wb-text-`, `--wb-radius-xs|--wb-radius-2|--wb-radius-3|--wb-radius-4|--wb-radius-xl|--wb-radius-full`, `--wb-shadow-xs|--wb-shadow-md|--wb-shadow-xl`, `--wb-z-`).
